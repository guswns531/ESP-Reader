# ESP-Reader Roadmap

## Product Goal

ESP-Reader는 `내부 플래시 기반 Markdown 문서 뷰어`로 시작해, 최종적으로는 `수식과 구조화된 기술 문서를 편하게 읽는 e-paper 기기`를 목표로 한다.

핵심 축은 아래 4가지다.

- Reading UX
- Markdown fidelity
- Math support
- Device stability

## v0.1 Reader Core

목표:

- 내부 플래시 기반 문서 목록
- 기본 읽기 상태머신
- 페이지 단위 읽기
- 위치 저장/복원
- 슬립/웨이크

포함 기능:

- `.md` 파일 스캔
- 문서 열기
- 다음/이전 페이지
- 진행률 표시
- 마지막 위치 저장

성공 기준:

- 장문 Markdown 문서를 끝까지 읽을 수 있다.
- 읽던 위치가 안정적으로 복원된다.

## v0.2 Markdown Basics

목표:

- Markdown 문서답게 보이기 시작하는 단계

포함 기능:

- `#`, `##`, `###`
- 문단
- 불릿/번호 리스트
- 구분선
- 인라인 코드
- 코드 블록
- 강조

성공 기준:

- 기술 문서와 노트 문서가 의미 있게 구분되어 보인다.

현재 상태:

- 구현됨
- 리스트, 코드블록, 수식 블록, 인용문, 구분선까지 기본 구조 구분 가능

## v0.3 Reading UX

목표:

- 단순히 읽히는 수준에서 `읽기 편한 수준`으로 끌어올린다.

포함 기능:

- 폰트 크기 조절
- 줄간격 조절
- 여백 조절
- 문서별 읽기 상태 저장
- 최근 문서
- 빠른 재진입
- 한글 줄바꿈 개선

성공 기준:

- 같은 문서를 여러 번 읽을 때 피로감이 줄어든다.

현재 상태:

- 부분 구현됨
- 문서별 마지막 페이지 저장
- 부분 갱신 기반 이동
- 수동 전체 refresh
- 한글 줄바꿈 보정

## v0.4 Structured Navigation

목표:

- 긴 문서 탐색성을 높인다.

포함 기능:

- 헤더 기반 섹션 점프
- 간단한 TOC
- 북마크 1~3개
- 문서 진행률과 남은 분량 감각 제공

성공 기준:

- 긴 기술 문서를 앞뒤로 이동하기 쉬워진다.

## v0.5 Math-Ready

목표:

- 수식이 포함된 문서를 읽을 수 있는 기반을 만든다.

포함 기능:

- inline math 구문 인식
- block math 구문 인식
- 수식 fallback 렌더링 정책
- 전처리 파이프라인 가능성 검토

기술 판단:

- `온디바이스 직접 렌더링`
- `사전 전처리 렌더링`

두 경로를 비교하고 하나를 선택한다.

성공 기준:

- 수식 문서가 최소한 맥락을 잃지 않고 읽힌다.

## v0.7 Math Viewer

목표:

- 수식이 텍스트 fallback이 아니라 실제 읽기 가능한 표현으로 보이기 시작한다.

포함 후보:

- 자주 쓰는 수식 패턴 렌더링
- 간단한 분수, 첨자, 블록 식 표현
- 수식 블록 캐시

## v1.0 Markdown Document Viewer

목표:

- ESP-Reader를 하나의 완성된 로컬 문서 뷰어로 만든다.

기대 상태:

- 읽기 UX가 안정적이다.
- Markdown subset이 충분히 실용적이다.
- 수식 문서도 제한 범위 안에서 읽을 수 있다.
- 장문 문서에서 위치 복원과 탐색이 자연스럽다.

## 지속 검토 항목

- 내부 플래시 용량 한계
- 폰트 용량과 렌더링 비용
- 수식 전략의 복잡도
- full refresh와 partial refresh의 균형
- Markdown parser를 어디까지 직접 구현할지 여부
